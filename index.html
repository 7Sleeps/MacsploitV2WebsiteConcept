Plugin API

Overview

Extend Macsploit V2 using the powerful Lua-based plugin API. You can create custom commands, interface elements, language support, and more.

Plugins are typically Lua scripts placed in the user's Macsploit V2 configuration directory (`~/.config/macsploitv2/plugins` or similar).

Example API Usage

Here's a simple example of registering a new command:

-- plugins/my_command.lua

-- Access the Macsploit V2 API namespace
local sv2 = macsploitv2

-- Register a command that shows an info message
sv2.commands.register("myplugin.showInfo", function()
  sv2.ui.show_info_message("My Plugin Command Executed!")
end)

-- Optionally, add it to the command palette/toolbar
-- (Specific API may vary)
sv2.ui.add_toolbar_button({
  command = "myplugin.showInfo",
  label = "Show Info",
  icon = "info-circle" -- Example icon name
})

sv2.log.info("My Plugin loaded successfully.")

This command might now appear as a button or be accessible via a command menu.

Available API Modules (Partial List)

  • macsploitv2.editor: Interact with the active editor (get/set text, selections, etc.).
  • macsplotv2.ui: Show notifications, manage UI elements.
  • macsploitv2.workspace: Access project files and folders.
  • macsploitv2.commands: Register and execute commands.
  • macsploitv2.log: Log messages to the developer console.
  • macsploitv2.connection: Manage connection status and port information.

Refer to the full API reference within the application (Help > API Documentation) for detailed information on all available modules and functions.
