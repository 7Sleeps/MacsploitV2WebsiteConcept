Plugin API

Overview

Extend Syntax V2 using the powerful Lua-based plugin API. You can create custom commands, interface elements, language support, and more.

Plugins are typically Lua scripts placed in the user's Syntax V2 configuration directory (`~/.config/syntaxv2/plugins` or similar).

Example API Usage

Here's a simple example of registering a new command:

-- plugins/my_command.lua

-- Access the Syntax V2 API namespace
local sv2 = syntaxv2

-- Register a command that shows an info message
sv2.commands.register("myplugin.showInfo", function()
  sv2.ui.show_info_message("My Plugin Command Executed!")
end)

-- Optionally, add it to the command palette
sv2.commands.add_to_palette({
  command = "myplugin.showInfo",
  title = "My Plugin: Show Info Message"
})

sv2.log.info("My Plugin loaded successfully.")

To run this command, open the command palette (e.g., `Ctrl+Shift+P`) and search for "My Plugin: Show Info Message".

Available API Modules (Partial List)

  • syntaxv2.editor: Interact with the active editor (get/set text, selections, etc.).
  • syntaxv2.ui: Show notifications, input boxes, quick picks.
  • syntaxv2.workspace: Access project files and folders.
  • syntaxv2.commands: Register and execute commands.
  • syntaxv2.log: Log messages to the developer console.

Refer to the full API reference within the application (Help > API Documentation) for detailed information on all available modules and functions.
